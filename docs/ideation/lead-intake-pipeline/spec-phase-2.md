# Implementation Spec: Lead Intake Pipeline — Phase 2

**Contract**: ./contract.md
**Estimated Effort**: M

## Technical Approach

Phase 2 makes Discord interactive. After Phase 1, n8n posts notifications via a Discord channel webhook (no buttons). Phase 2 swaps that for a Discord application + bot — the bot posts the message with `Approve / Edit / Discard` buttons, Discord POSTs every button click to a Worker route, and the Worker writes the action into D1 for n8n to pick up on the next poll cycle.

Three concrete moves:

1. **Add a `lead_actions` table** to `src/db/schema.ts`. Run drizzle-kit to generate the next migration (`migrations/0001_*.sql`). Stores every button click (lead_id, action, user_id, ts) with `processed=false`.
2. **Add a `/api/discord/interact` Worker route** that verifies Discord's `Ed25519` signature on every request, ACKs within 3 seconds, and writes the click to D1. Modal interactions (the "Edit" flow) get the same treatment — the modal submission lands as a `lead_actions` row with the edited body.
3. **Reuse the pull pattern.** New endpoint `/api/lead-actions/pending` is structurally identical to Phase 1's `/api/leads/pending` — bearer auth, claim-on-read via Drizzle.

The Discord application setup (registering the app, creating the bot, configuring the Interactions Endpoint URL) happens in the Discord Developer Portal — not in code. Codebase only needs to expose the verified endpoint, persist actions, and let n8n consume them.

## Feedback Strategy

**Inner-loop command**: `bun run check`

**Playground**: Astro dev server (`bun run dev`) plus a forwarding tunnel for Discord to reach localhost during dev. Use `cloudflared tunnel --url http://localhost:4321` to give Discord a reachable Interactions Endpoint URL during development. `bun run db:studio` for inspecting `lead_actions` rows after button clicks.

**Why this approach**: Signature verification and the 3-second ACK contract are the two things most likely to break. Both can be tested by Discord's "Save Changes" validation step in the Developer Portal — Discord sends a synthetic PING and expects a `type: 1` response within 3 seconds. That single click validates the signature path end-to-end.

### 1. Add `leadActions` table to schema

**File**: `src/db/schema.ts`

Append to the existing schema file (don't replace — Phase 1's `leads` table stays):

```typescript
export const leadActions = sqliteTable(
  'lead_actions',
  {
    id: text('id').primaryKey(),                          // uuid v4
    leadId: text('lead_id').notNull(),                    // refs leads.id, not enforced
    action: text('action', {
      enum: ['approve', 'edit', 'discard'],
    }).notNull(),
    editedBody: text('edited_body'),                      // only set when action='edit'
    discordUserId: text('discord_user_id').notNull(),
    discordInteractionId: text('discord_interaction_id').notNull(),
    processed: integer('processed', { mode: 'boolean' })  // SQLite has no bool; mode flag handles it
      .notNull()
      .default(false),
    createdAt: text('created_at')
      .notNull()
      .default(sql`(datetime('now'))`),
    processedAt: text('processed_at'),
  },
  (table) => ({
    interactionIdx: uniqueIndex('idx_lead_actions_interaction').on(table.discordInteractionId),
    processedIdx: index('idx_lead_actions_processed').on(table.processed, table.createdAt),
    leadIdx: index('idx_lead_actions_lead').on(table.leadId),
  }),
);

export type LeadAction = typeof leadActions.$inferSelect;
export type NewLeadAction = typeof leadActions.$inferInsert;
```

You'll need to also add `uniqueIndex` to the existing import:

```typescript
import { sqliteTable, text, integer, index, uniqueIndex } from 'drizzle-orm/sqlite-core';
```

**Key decisions**:
- `discordInteractionId` is `UNIQUE` so duplicate interaction deliveries (rare but possible) are deduped at the DB layer. The Drizzle helper handles the constraint violation gracefully and returns `{ inserted: false }`.
- `processed` is a real boolean using Drizzle's `mode: 'boolean'` (stored as 0/1 under the hood; surfaced as `boolean` in TS).
- No FK on `leadId` — D1's SQLite doesn't enforce them by default. We trust the client.

### 3. Lead-actions Drizzle helpers

**File**: `src/lib/lead-actions.ts`

**Pattern to follow**: `src/lib/leads.ts` from Phase 1 (parallel structure).

```typescript
import { and, asc, eq, inArray, sql } from 'drizzle-orm';
import type { D1Database } from '@cloudflare/workers-types';
import { getDb } from '../db/client';
import { leadActions, type LeadAction } from '../db/schema';

export type LeadActionKind = LeadAction['action'];

export async function insertLeadAction(
  d1: D1Database,
  input: {
    id: string;
    leadId: string;
    action: LeadActionKind;
    editedBody: string | null;
    discordUserId: string;
    discordInteractionId: string;
  },
): Promise<{ inserted: boolean }> {
  const db = getDb(d1);
  try {
    await db.insert(leadActions).values({
      id: input.id,
      leadId: input.leadId,
      action: input.action,
      editedBody: input.editedBody,
      discordUserId: input.discordUserId,
      discordInteractionId: input.discordInteractionId,
    });
    return { inserted: true };
  } catch (err) {
    // UNIQUE on discord_interaction_id is the idempotency path
    if (String(err).includes('UNIQUE')) return { inserted: false };
    throw err;
  }
}

export async function claimPendingActions(
  d1: D1Database,
  limit = 50,
): Promise<LeadAction[]> {
  const db = getDb(d1);
  const pending = await db
    .select({ id: leadActions.id })
    .from(leadActions)
    .where(eq(leadActions.processed, false))
    .orderBy(asc(leadActions.createdAt))
    .limit(limit);
  if (pending.length === 0) return [];
  const ids = pending.map((row) => row.id);
  const claimed = await db
    .update(leadActions)
    .set({ processed: true, processedAt: sql`(datetime('now'))` })
    .where(inArray(leadActions.id, ids))
    .returning();
  return claimed;
}
```

### 4. Discord helpers (Ed25519 + types)

**File**: `src/lib/discord.ts`

```typescript
// Discord Interactions Endpoint contract:
// https://discord.com/developers/docs/interactions/receiving-and-responding

export const InteractionType = {
  PING: 1,
  APPLICATION_COMMAND: 2,
  MESSAGE_COMPONENT: 3,
  APPLICATION_COMMAND_AUTOCOMPLETE: 4,
  MODAL_SUBMIT: 5,
} as const;

export const InteractionResponseType = {
  PONG: 1,
  CHANNEL_MESSAGE_WITH_SOURCE: 4,
  DEFERRED_CHANNEL_MESSAGE_WITH_SOURCE: 5,
  DEFERRED_UPDATE_MESSAGE: 6,
  UPDATE_MESSAGE: 7,
  MODAL: 9,
} as const;

export interface DiscordInteraction {
  id: string;
  type: number;
  data?: {
    custom_id?: string;
    component_type?: number;
    components?: Array<{
      type: number;
      components?: Array<{ type: number; custom_id: string; value: string }>;
    }>;
  };
  member?: { user?: { id: string } };
  user?: { id: string };
  message?: { id: string };
}

/**
 * Verify a Discord interaction request signature using the application's
 * Ed25519 public key. Discord requires this on every interaction; failing
 * verification means returning 401.
 *
 * Workers runtime supports SubtleCrypto.importKey + verify with Ed25519
 * given a recent compatibility_date. No npm dep needed.
 */
export async function verifyDiscordSignature(
  request: Request,
  publicKeyHex: string,
): Promise<{ verified: true; body: string } | { verified: false }> {
  const signature = request.headers.get('x-signature-ed25519');
  const timestamp = request.headers.get('x-signature-timestamp');
  if (!signature || !timestamp) return { verified: false };

  const body = await request.text();
  const message = new TextEncoder().encode(timestamp + body);
  const sig = hexToBytes(signature);
  const key = hexToBytes(publicKeyHex);

  let cryptoKey: CryptoKey;
  try {
    cryptoKey = await crypto.subtle.importKey(
      'raw',
      key,
      { name: 'Ed25519' },
      false,
      ['verify'],
    );
  } catch {
    return { verified: false };
  }

  const ok = await crypto.subtle.verify('Ed25519', cryptoKey, sig, message);
  return ok ? { verified: true, body } : { verified: false };
}

function hexToBytes(hex: string): Uint8Array {
  const out = new Uint8Array(hex.length / 2);
  for (let i = 0; i < out.length; i++) {
    out[i] = parseInt(hex.slice(i * 2, i * 2 + 2), 16);
  }
  return out;
}

/**
 * Build the modal payload for the "Edit draft" flow.
 */
export function buildEditModal(leadId: string, draft: string) {
  return {
    type: InteractionResponseType.MODAL,
    data: {
      custom_id: `edit:${leadId}`,
      title: 'Edit draft',
      components: [
        {
          type: 1, // ACTION_ROW
          components: [
            {
              type: 4, // TEXT_INPUT
              custom_id: 'draft_body',
              label: 'Draft reply',
              style: 2, // PARAGRAPH
              required: true,
              min_length: 10,
              max_length: 4000,
              value: draft.slice(0, 4000),
            },
          ],
        },
      ],
    },
  };
}
```

### 5. Discord interactions endpoint

**File**: `src/pages/api/discord/interact.ts`

```typescript
import type { APIRoute } from 'astro';
import { eq } from 'drizzle-orm';
import { getDb } from '../../../db/client';
import { leads } from '../../../db/schema';
import { getEnv } from '../../../lib/leads';
import {
  buildEditModal,
  InteractionResponseType,
  InteractionType,
  verifyDiscordSignature,
  type DiscordInteraction,
} from '../../../lib/discord';
import { insertLeadAction, type LeadActionKind } from '../../../lib/lead-actions';

export const prerender = false;

function jsonResponse(status: number, body: unknown): Response {
  return new Response(JSON.stringify(body), {
    status,
    headers: { 'content-type': 'application/json' },
  });
}

function ephemeralReply(content: string) {
  return {
    type: InteractionResponseType.CHANNEL_MESSAGE_WITH_SOURCE,
    data: { content, flags: 64 }, // EPHEMERAL — only the clicker sees it
  };
}

function isLeadAction(s: string | undefined): s is LeadActionKind {
  return s === 'approve' || s === 'edit' || s === 'discard';
}

function actionAck(action: LeadActionKind): string {
  switch (action) {
    case 'approve': return 'Approved. Sending now.';
    case 'discard': return 'Discarded. No reply will be sent.';
    case 'edit': return 'Opening editor…'; // unused; edit hits the modal path
  }
}

export const POST: APIRoute = async ({ request, locals }) => {
  const env = getEnv(locals) as
    | (ReturnType<typeof getEnv> & { DISCORD_PUBLIC_KEY?: string })
    | null;
  if (!env || !env.DISCORD_PUBLIC_KEY || !env.LEADS_DB) {
    return new Response('not configured', { status: 500 });
  }

  const verification = await verifyDiscordSignature(request, env.DISCORD_PUBLIC_KEY);
  if (!verification.verified) {
    return new Response('invalid request signature', { status: 401 });
  }

  let interaction: DiscordInteraction;
  try {
    interaction = JSON.parse(verification.body) as DiscordInteraction;
  } catch {
    return new Response('invalid json', { status: 400 });
  }

  // Discord PING for endpoint validation
  if (interaction.type === InteractionType.PING) {
    return jsonResponse(200, { type: InteractionResponseType.PONG });
  }

  const userId = interaction.member?.user?.id ?? interaction.user?.id ?? 'unknown';

  // Button click: custom_id format "approve:<leadId>" | "edit:<leadId>" | "discard:<leadId>"
  if (interaction.type === InteractionType.MESSAGE_COMPONENT) {
    const customId = interaction.data?.custom_id ?? '';
    const [action, leadId] = customId.split(':');
    if (!leadId || !isLeadAction(action)) {
      return jsonResponse(200, ephemeralReply("Couldn't parse that button. Reload Discord."));
    }

    if (action === 'edit') {
      // Pre-fill modal with the current draft via Drizzle.
      const db = getDb(env.LEADS_DB);
      const [lead] = await db
        .select({ draft: leads.draft })
        .from(leads)
        .where(eq(leads.id, leadId))
        .limit(1);
      return jsonResponse(200, buildEditModal(leadId, lead?.draft ?? ''));
    }

    await insertLeadAction(env.LEADS_DB, {
      id: crypto.randomUUID(),
      leadId,
      action,
      editedBody: null,
      discordUserId: userId,
      discordInteractionId: interaction.id,
    });

    return jsonResponse(200, ephemeralReply(actionAck(action)));
  }

  // Modal submit: custom_id is "edit:<leadId>"; body is in components[0].components[0].value
  if (interaction.type === InteractionType.MODAL_SUBMIT) {
    const customId = interaction.data?.custom_id ?? '';
    const [, leadId] = customId.split(':');
    const body = interaction.data?.components?.[0]?.components?.[0]?.value ?? '';
    if (!leadId || body.length < 10) {
      return jsonResponse(200, ephemeralReply('That edit looks empty.'));
    }

    await insertLeadAction(env.LEADS_DB, {
      id: crypto.randomUUID(),
      leadId,
      action: 'edit',
      editedBody: body,
      discordUserId: userId,
      discordInteractionId: interaction.id,
    });

    return jsonResponse(200, ephemeralReply('Got it. Sending the edited reply.'));
  }

  return jsonResponse(200, { type: InteractionResponseType.PONG });
};
```

### 6. Lead-actions pull endpoint

**File**: `src/pages/api/lead-actions/pending.ts`

**Pattern to follow**: `src/pages/api/leads/pending.ts` from Phase 1 (parallel structure, same auth shape).

```typescript
import type { APIRoute } from 'astro';
import { getEnv } from '../../../lib/leads';
import { claimPendingActions } from '../../../lib/lead-actions';

export const prerender = false;

function isAuthorized(request: Request, expected: string | undefined): boolean {
  if (!expected) return false;
  const header = request.headers.get('authorization');
  return Boolean(header?.startsWith('Bearer ') && header.slice(7).trim() === expected);
}

export const GET: APIRoute = async ({ request, locals }) => {
  const env = getEnv(locals);
  if (!env || !isAuthorized(request, env.WORKER_PULL_TOKEN)) {
    return new Response(JSON.stringify({ error: 'unauthorized' }), {
      status: 401,
      headers: { 'content-type': 'application/json' },
    });
  }
  const url = new URL(request.url);
  const limit = Math.min(
    Math.max(Number.parseInt(url.searchParams.get('limit') ?? '50', 10) || 50, 1),
    200,
  );
  const actions = await claimPendingActions(env.LEADS_DB, limit);
  return new Response(JSON.stringify({ actions }), {
    status: 200,
    headers: { 'content-type': 'application/json', 'cache-control': 'no-store' },
  });
};
```

## API Design

| Method | Path | Auth | Purpose |
|---|---|---|---|
| `POST` | `/api/discord/interact` | Discord Ed25519 signature | Receive button clicks + modal submits |
| `GET` | `/api/lead-actions/pending` | `Bearer $WORKER_PULL_TOKEN` | n8n claims pending actions |

## Failure Modes

| Component | Failure | Behavior |
|---|---|---|
| `verifyDiscordSignature` | Missing or invalid signature | 401, request never reaches business logic |
| `verifyDiscordSignature` | `crypto.subtle.importKey` fails (runtime missing Ed25519) | 401 (logged); deploy must use a recent compatibility_date |
| `discord/interact` | `insertLeadAction` hits `UNIQUE` violation | Treated as idempotent retry; ack returned, no row inserted |
| `discord/interact` | Modal value > 4000 chars | Discord client validates; if it slips through, the next-row write succeeds (max_length is advisory at the API layer too) |
| `lead-actions/pending` | D1 query fails | 500; n8n retries on next poll |
| n8n applies an action | Lead already in `outcome=approved` | n8n decides the conflict policy; codebase doesn't care |
| Discord retries the same interaction | Second POST | UNIQUE on `discord_interaction_id` rejects insert; we still return 200 |

6. Register a Discord application at https://discord.com/developers/applications
7. Copy Public Key + App ID into Cloudflare secrets:
   - `bunx wrangler secret put DISCORD_PUBLIC_KEY`
   - `bunx wrangler secret put DISCORD_APP_ID`
8. Set the application's "Interactions Endpoint URL" to `https://birdcar.dev/api/discord/interact` — Discord sends a verification PING; if the endpoint validates and PONGs within 3s, Discord saves the URL.
9. From n8n, build a test message with three buttons (`approve:test-lead-id`, `edit:test-lead-id`, `discard:test-lead-id`) and post to `#leads`.
10. Click each button. Verify rows appear in `lead_actions` (use `bun run db:studio`) with the right `action` values. Click the same button twice — second click should be deduped via UNIQUE.
11. Click "Edit", verify modal opens pre-filled with the lead's existing draft. Submit modal with a new body. Verify a `lead_actions` row with `action='edit'` and `edited_body` set.
12. `curl -H "Authorization: Bearer $TOKEN" https://birdcar.dev/api/lead-actions/pending` → returns the unprocessed actions, marks them processed.
13. `bun run build:ci` → 0 errors.
