## Skill Architecture

**Primary pattern**: Context-aware (decision table: what asset type?) + Sequential (within each generation target).

The SKILL.md presents a decision table when triggered: favicon / OG / per-post OG / manifest / all. Based on selection, it runs the corresponding uv script(s) via Bash. Each script is self-contained with embedded `uv` dependency declarations (Pillow, etc.).

Brand constants live in `references/brand.md` — the SKILL.md references this doc, and scripts hardcode the same values (no dynamic config loading). This dual-source is acceptable because the palette is stable and scripts need constants at the Python level.

No agents needed. All work is deterministic script execution. The skill uses AskUserQuestion for input selection and Bash for script execution.

## Per-Component Details

### SKILL.md

- **Purpose**: Entry point. Trigger matching, workflow orchestration, script invocation.
- **Key behaviors**:
  - Decision table for asset type selection via AskUserQuestion
  - Script invocation patterns for each asset type with all flag variations
  - Output delivery: show generated files, emit HTML meta tag snippets
  - Brand awareness: references `references/brand.md` for context
  - Error handling: check script exit codes, report failures clearly

### references/brand.md

- **Purpose**: Single source of truth for birdcar.dev brand system
- **Key behaviors**:
  - Catppuccin Mocha palette with hex codes and semantic mappings
  - Font stack: Archivo (headings, 700), Space Grotesk (body, 400), JetBrains Mono (code, 400)
  - OG image layout spec: title position, description position, accent strip, margins
  - Color usage rules: which colors for backgrounds, text, accents

### references/platforms.md

- **Purpose**: Platform-specific output specifications
- **Key behaviors**:
  - Favicon sizes: 16x16, 32x32, 96x96, 180x180 (apple-touch), 192x192, 512x512, ICO (multi-res)
  - OG sizes: 1200x630 (Facebook/LinkedIn), 1200x675 (Twitter), 1200x1200 (square)
  - Format requirements per platform
  - File naming conventions

### scripts/generate_favicons.py

- **Purpose**: Generate complete favicon set from various input sources
- **Key behaviors**:
  - uv script header with Pillow dependency
  - Input modes: `--svg <path>`, `--text <char>`, `--emoji <char>`, `--image <path>`
  - Optional: `--bg-color` for background override (defaults to brand base)
  - Output: all favicon sizes + ICO to specified output directory
  - SVG rasterization via cairosvg (embedded dep)
  - Text/emoji rendering with brand fonts

### scripts/generate_og.py

- **Purpose**: Generate site-wide OG images with brand typography
- **Key behaviors**:
  - uv script header with Pillow dependency
  - Input: `--title <text>` and optional `--subtitle <text>`
  - Layout: title in Archivo 700, subtitle in Space Grotesk 400, accent strip, Catppuccin Mocha bg
  - Output: og-image.png (1200x630), twitter-image.png (1200x675), og-square.png (1200x1200)
  - Font auto-download from Google Fonts if not cached

### scripts/generate_post_og.py

- **Purpose**: Batch generate per-post OG images from blog frontmatter
- **Key behaviors**:
  - uv script with Pillow + PyYAML dependencies
  - Reads `src/content/blog/*.md`, extracts title + description from frontmatter
  - Generates `public/og/{slug}.png` (1200x630) for each post
  - Layout: post title in Archivo, description in Space Grotesk, accent color strip, dark bg
  - Skip posts that already have an `image` field set in frontmatter
  - `--force` flag to regenerate all

### scripts/generate_manifest.py

- **Purpose**: Generate PWA manifest.json
- **Key behaviors**:
  - uv script (no image deps needed)
  - Outputs `public/manifest.json` with icon references, theme_color, background_color from brand
  - References favicon paths generated by generate_favicons.py
